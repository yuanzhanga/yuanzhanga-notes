画布组件布局（绝对定位）转为 实际导出的 Flex 布局：每个组件都会有详细的坐标信息、宽度和好度，利用这些信息去分析组件之间是否属于同一行，如果是同一行，将会用一个弹性容器去包裹他们，而组件之间的间距，用 margin 值替换

多组件的组合:鼠标框选覆盖组件的各种情况

组件从物料区拖至画布及画布状态存储管理：对于用户在编辑页正在操作的内容，会将数据保存下来，在刷新时或者二次使用时，不必再重复识别或者重新搭建，节省时间提高效率。同时，画布提供完善的撤销、重做等状态记录管理
    给组件添加 draggble 属性，使其可以拖拽；
    在可拖拽组件物料区的容器上添加对 dragstart 事件的监听，它主要用于获取当前被拖拽的组件信息；
    在画布区添加对 drop 事件的监听，在拖拽结束时（释放鼠标）触发，用于在接收拖拽的组件信息；
    在 drop 事件中获取传输过来的组件唯一的 key ，通过这个 key 值在准备好的模版数据中获取相应的组件数据；
    将该数据添加到 vuex 中的 画布 data 中，并且渲染相应的组件
画布上的内容是通过数据搭配业务组件进行渲染的，数据保存在一个数组中，那就可以使用二维数组对画布状态进行持续的保存。
当画布数据进行改变时（组件新建删除，移动放大缩小等），就推进二维数组进行保存，通过 index 可以获取某一时刻的画布状态

数据处理
整体上，用户点击画布组件，选中当前编辑组件，编辑 UI 展示组件编辑的 组件属性 和 容器属性。
用户编辑过程，可以对属性进行 增加，删除，修改。不同类型的属性会有不同控件支持灵活编辑。
之后，用户刷新画布，更新当前编辑完成组件，画布刷新组件依照新的属性重新 render 组件
